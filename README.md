# 说文 saynow

说文是一个 Windows 桌面端语音输入助手。它的使用方式很简单：在任意输入框里按住快捷键说话，松开后自动识别语音，并把文字写回当前输入位置。

说文更接近键盘输入法旁边的一个轻量助手：常驻托盘，随叫随用，服务于写代码、聊天、记笔记和处理文档时的高频输入。

## 为什么做它

通用语音识别通常能听懂大意，而每个人都有自己的表达习惯。项目名、英文缩写、技术字段、人名、常用短语、数字格式，以及你希望输出更口语还是更书面，这些东西都值得被单独记住。

说文把重点放在个性化输入上：

- 你可以维护自己的专属词库，让模型优先识别那些对你重要的词。
- 你可以设置风格提示词，让输出更符合你的写作场景，例如聊天、会议纪要、工单描述或技术记录。
- 你可以在识别后纠正结果，应用会把明确纠错和成功识别历史整理成个人偏好。
- 自学习引擎会在后台提炼稳定规则，并在之后的识别请求中作为“用户个性化识别偏好”参与上下文。

简单说，它把语音、场景和个人偏好放在一起处理，输出更接近你真正想输入的文字。

## 现在能做什么

说文当前已经打通了桌面语音输入的主流程：

- 全局快捷键录音：在任意应用中按住快捷键开始说话，松开结束。
- 自动写入文本：识别完成后写入当前聚焦输入框；写入失败时降级复制到剪贴板。
- 托盘常驻：关闭窗口后仍可在后台运行，适合日常输入场景。
- 多模型配置：内置 MiMo-v2.5、MiMo ASR 和 Qwen3.5-Omni 配置模板，也支持自定义 OpenAI-compatible Base URL、模型和 API Key。
- 专属词库：维护常用术语、英文缩写、项目名和容易误识别的词。
- 风格提示词：用自然语言定义输出风格，让识别结果更贴近具体场景。
- 纠错与撤销：识别后可编辑刚才输入的内容，并把纠错记录保存为学习样本。
- 自学习引擎：可配置独立模型，把纠错记录和成功识别历史整理成个性化规则。
- 历史与统计：查看最近识别记录、使用次数和输入效率数据。

## 产品亮点

说文的核心亮点是“越用越像你”。

手动个性化解决的是明确偏好：哪些词一定要识别对，输出应该是什么语气，技术词是否保留英文，会议内容是否整理成更书面的表达。

自学习引擎解决的是长期偏好：当前实现会把明确纠错视为强信号，把成功识别历史视为弱信号，在应用空闲时批量整理，生成小而稳定的候选规则。后续识别时，这些规则会进入提示上下文，帮助模型更好地处理你的专有名词、格式习惯、口头表达和数字语境。

这种设计让说文在模型能力之外，建立一层属于用户自己的输入记忆。

## 适合的场景

- 在聊天软件里快速输入长句，保持当前输入节奏。
- 写技术文档、Issue、PR 描述时，保留英文术语和字段名的准确形式。
- 记录会议或想法时，希望口语内容自动整理得更清楚。
- 经常输入固定项目名、产品名、人名或缩写，希望识别稳定记住。
- 接入多家供应商的多模态语音模型，同时保留统一的桌面输入体验。

当前版本聚焦 Windows 桌面上的快捷语音输入、个性化识别和本地使用记录。

## 开发

安装依赖并启动前端开发服务：

```bash
npm install
npm run dev
```

常用验证命令：

```bash
npm test -- --run
npm run build
cd src-tauri && cargo test
```

Windows 桌面能力需要在 Windows 原生环境验证：

```bash
npm run tauri dev -- --features desktop
```

需要重点验证的桌面能力包括托盘常驻、全局快捷键、麦克风录音、当前聚焦窗口捕获、文本自动写入，以及写入失败时复制到剪贴板的降级路径。

## 技术实现

说文使用 Vue 3、TypeScript 和 Vite 构建前端界面；桌面端基于 Tauri 2；本地能力和数据逻辑主要由 Rust 实现。

后端负责本地命令、SQLite 持久化、模型 Provider 调用、Windows 文本写入，以及个性化上下文构建。SQLite 保存供应商配置、快捷键配置、识别记录、纠错记录、词库、风格提示词、学习引擎配置和学习规则。模型调用以 OpenAI-compatible Chat Completions 为基础，供应商差异主要在 `src-tauri/src/provider.rs` 中适配。

自学习相关逻辑集中在 Rust 后端的数据命令和学习模块中。当前学习闭环包括：保存 `rawText -> correctedText` 纠错对、记录成功识别历史、按空闲时间或手动触发整理、请求配置的学习模型、解析 JSON 学习规则、写入本地规则表，并在后续识别时把可用规则拼入个性化提示上下文。
